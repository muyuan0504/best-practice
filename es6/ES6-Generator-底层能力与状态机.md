# ES6 为 Generator 提供了哪些“底层能力”，为何不必再用手写状态机？

## 先澄清一句

**“不必再用状态机模拟”**指的是：在 **ES6 引擎原生支持** 的前提下，**你（或编译器）不必再把 `yield` 翻译成 `switch(state)` 那种显式状态机**。  
在 **ES5 垫片**（如 Babel + `regenerator-runtime`）里，**仍然**是状态机，只是由编译器生成，而不是你手写。

也就是说：省掉状态机的是 **原生 `function*` + `yield` 的运行时实现**，不是又发明了另一套与 Generator 无关的 API。

---

## ES5 为什么只能用手写（或编译出的）状态机？

ES5 语言与典型引擎能力里缺少这几样（见同目录 `异步函数/es5-generator.js` 中的注释）：

1. **没有“在函数中间暂停、之后再从同一位置继续”的语义**；函数一 `return` 就结束，局部变量栈帧销毁。
2. **没有标准的“惰性拉取”协议**；要自己约定 `next()` 返回 `{ value, done }`。
3. 因此 **regenerator** 一类方案只能：**用 `state` 表示执行到第几个 `yield`**，用 `switch` 把控制流摊平，用闭包保存变量——这就是状态机。

---

## ES6 新增的、与 Generator 直接相关的“底层能力”

下面按 **规范语义 → 协议 → 消费端 → 引擎实现** 分层说明。

### 1. 语言层面的生成器语义（规范要求引擎实现）

- **`function*` 与 `yield` / `yield*`**：把“可暂停、可恢复、可向内部注入值”的协程式控制流写进语法。
- **生成器对象即迭代器**：调用 `g = gen()` 得到的对象自带 `next`（以及可选的 `return` / `throw`），且符合迭代器结果形状。
- **`yield` 与 `next(arg)` 的传参约定**：暂停点可以接收外界传入的值，这是状态机里要额外编码的细节，原生由引擎处理。

这些是 **ECMAScript 2015（ES6）规范中的 Generator 章节**定义的能力；没有它们，就只能用状态机在用户代码层模拟。

### 2. 标准迭代协议（Iterator / Iterable）

- **`Symbol.iterator` 与可迭代协议**：统一“怎么拿到一个迭代器”。
- **迭代器协议**：`next()` 返回 `{ value, done }`（及完成后的行为）。

Generator 实例 **天然满足迭代器协议**，并且其 **原型链上的 `Symbol.iterator`** 行为由规范规定，因此 **`for...of`、展开、解构赋值** 等可以一致地对待“生成器产出的序列”，而不必每个库各写一套。

### 3. 消费端语法：`for...of` 等

- **`for...of`**：对 **Iterable** 做语法糖，内部反复 `iterator.next()` 直到 `done: true`。
- 与 **展开运算符 `...`、数组解构** 等配合时，右侧若是可迭代对象（包括 generator 返回值），由引擎按协议消费。

这些不是 Generator 的“实现细节”，但它们是 **ES6 把迭代从库约定提升为语言基础设施** 的一部分，与 Generator 同属一块拼图。

### 4. 引擎 / 运行时真正“替代状态机”的部分（虚拟机层）

手写状态机的本质，是用 **`state` 变量 + 分支** 模拟 **“执行指针停在哪一条 `yield` 上”**。  
ES6 引擎对 Generator 的典型实现思路是（概念上，非绑定某一引擎源码）：

- 为每个生成器实例维护 **独立的执行上下文**（可理解为冻结的栈帧或类似的 continuation 结构）：
  - **停在哪一条 `yield`（程序计数 / resume 点）**
  - **局部绑定（`let`/`const`/参数）与闭包环境**
  - **异常与 `try/finally` 相关的完成状态**（比三行教程版状态机复杂得多）
- 每次 `next()` / `throw()` 时 **恢复该上下文**，跑到下一个 `yield` 或 `return`，再挂起。

也就是说：**“第几个状态”不再暴露为你代码里的 `switch (state)`**，而是由 **VM 内部的生成器状态机（或等价结构）** 维护。  
从**应用开发者视角**看，就是“不再需要手写那层状态机”。

---

## 小结对照

| 维度 | ES5 模拟 | ES6 原生 |
|------|----------|----------|
| 暂停 / 恢复 | 用 `state` + `switch` 编码位置 | 引擎保存生成器执行位置与局部环境 |
| 迭代形状 | 自行约定或模仿 `{ value, done }` | 规范化的 Iterator / Iterable |
| 消费方式 | 手写循环调 `next()` | `for...of`、展开、解构等与协议统一 |
